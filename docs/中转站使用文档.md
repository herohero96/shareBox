# 中转站使用文档

> 给新用户看的快速上手说明：如何注册、充值/兑换额度、创建令牌，以及在各种 AI 客户端里使用我们的中转站。

---

## 1. 这个中转站是什么？

这是一个基于 **New API** 搭建的 AI 接口中转站。

你可以把它理解成：

```text
你的客户端 / 代码
    ↓
我们的中转站
    ↓
MiniMax 官方模型 API
```

你不用直接配置 MiniMax 官方 Key，只需要使用本站生成的 **API 令牌**，就可以通过 OpenAI 兼容格式调用 MiniMax 模型。

---

```text
http://51.21.252.41:3000
```

### API 接口地址

```text
http://51.21.252.41:3000/v1
```

---

## 3. 新用户使用流程

### 第一步：注册账号

打开注册页面：

```text
http://51.21.252.41:3000/register
```

填写用户名和密码完成注册。

---

### 第二步：充值或兑换额度

登录后进入后台，根据管理员提供的方式进行：

- 兑换码兑换
- 管理员手动充值
- 其他充值方式

兑换后可以在账号页面看到自己的剩余额度。

---

### 第三步：创建 API 令牌

登录后台后：

```text
令牌 → 添加令牌 / 新建令牌
```

建议填写：

```text
名称：我的测试令牌
额度：按需填写，或者使用账号额度
过期时间：不设置 / 长期有效
可用分组：default
可用模型：默认即可
```

创建完成后，复制 `sk-` 开头的完整 Key，例如：

```text
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> 注意：调用 API 时必须使用「令牌 Key」，不是兑换码，也不是注册链接。

---

## 4. 客户端应该怎么填？

如果你的客户端支持 **OpenAI Compatible / OpenAI 兼容 / 自定义 OpenAI**，就这样填写：

```text
Provider / 类型：OpenAI Compatible / 自定义 OpenAI
Base URL / API 地址：http://51.21.252.41:3000/v1
API Key / 密钥：你的 sk- 开头令牌
Model / 模型：MiniMax-M2.7
```

推荐默认模型：

```text
MiniMax-M2.7
```

如果想要更快，可以试：

```text
MiniMax-M2.7-highspeed
```

## 6. 在常见客户端中使用

### Cherry Studio

1. 打开 Cherry Studio
2. 添加供应商
3. 选择 **OpenAI 兼容** 或 **自定义 OpenAI**
4. 填写：

```text
API 地址：http://51.21.252.41:3000/v1
API Key：你的 sk- 令牌
模型：MiniMax-M2.7
```

---

### LobeChat / NextChat / OpenCat / 其他客户端

只要支持 OpenAI 兼容接口，都按照下面配置：

```text
Base URL：http://51.21.252.41:3000/v1
API Key：你的 sk- 令牌
Model：MiniMax-M2.7
```

## 7. 代码调用示例

### curl 示例

```bash
curl http://51.21.252.41:3000/v1/chat/completions \
  -H "Authorization: Bearer sk-你的令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MiniMax-M2.7",
    "messages": [
      {"role": "user", "content": "你好，请介绍一下你自己"}
    ]
  }'
```

---

### Python 示例

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://51.21.252.41:3000/v1",
    api_key="sk-你的令牌"
)

response = client.chat.completions.create(
    model="MiniMax-M2.7",
    messages=[
        {"role": "user", "content": "你好"}
    ]
)

print(response.choices[0].message.content)
```

---

### Node.js 示例

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'http://51.21.252.41:3000/v1',
  apiKey: 'sk-你的令牌'
});

const response = await client.chat.completions.create({
  model: 'MiniMax-M2.7',
  messages: [
    { role: 'user', content: '你好' }
  ]
});

console.log(response.choices[0].message.content);
```

## 8. 常见错误

### Invalid token

说明 API Key 不正确。

请检查：

- 是否复制了完整的 `sk-` 令牌
- 是否把兑换码误当成了 API Key
- 令牌是否被禁用或删除

---

### model_price_error

说明模型价格或自用模式配置有问题。

普通用户无需处理，联系管理员即可。

---

### 404 / 无法连接

请检查 API 地址是否填写正确：

```text
http://51.21.252.41:3000/v1
```

不要漏掉 `/v1`。
